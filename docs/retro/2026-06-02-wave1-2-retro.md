## 2. 關鍵事件

### 2.1 Catastrophic Bug 攔截（PR #252）

Reviewer 在 boot validation 優化中發現 `REQUIRED_KEYS` 漏掉 `cashSessions`。若沒攔到，所有升級到 v2 的使用者 localStorage state 會被 `validatePersistedState` 拒絕，觸發 store reset 到 initial mock data——**全部使用者資料遺失**。

同 PR 還發現 double-migration 衝突（Zustand migrate middleware + 手動 migrateState）和 migrateState 對 v2+ 完全不驗證結構。

**教訓**: 修改 validation/migration pipeline 必須：
- 有完整的 migration dependency graph
- 驗證所有 `REQUIRED_KEYS`、`ARRAY_KEYS`、`OBJECT_KEYS` 與 `PosState` interface 一致
- 建立 migration regression test checklist

### 2.2 Firebase Bundle Tradeoff（PR #253）

從 418KB 降到初始 **28KB**（-93%），但 dynamic import 失去 tree-shaking 使總量增至 502KB。Tradeoff 合理但需監控 lazy chunk 是否確實在需要的路徑才載入。

### 2.3 Impl 卡死與 Nudge 恢復

Impl 兩次因 ctx:100% + API Error（proxy `InvalidHTTPResponse`）卡死。Nudge 有效恢復，但若發生在 pre-impl spike 期間可能錯過 dispatch nuance。

### 2.4 Watchdog 部署

從 CronCreate ack（錯誤方式）改為 daemon schedule + dual-snapshot diff（正確方式），每 10 分鐘檢查 impl + reviewer。ctx:100% 判定不準確，已排除。
