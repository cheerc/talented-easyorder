## Task 3: Fix useFocusSync to preserve view-status focusZone

**Files:**
- Modify: `frontend/src/hooks/useFocusSync.ts:17-21`

### Step 3: Prevent useFocusSync from overwriting view-status

The current `useFocusSync` (line 19-21) unconditionally sets `setFocusZone('mode-' + state.mode)` when `state.kind === 'student_selected'`. This races with `setFocusZone('view-status')` set in PosColumn's `onStudentClick` — both run in the same render cycle, but `useFocusSync`'s `useEffect` fires after render, overwriting the 'view-status' value.

The fix: **pass `focusZone` into `useFocusSync`** and skip the sync when `focusZone` is already `'view-status'`.

⚠️ **Alternative considered:** We could instead just not sync when `student_selected` — but that would break Q/W keyboard mode switching which relies on `useFocusSync` to update `focusZone` when `state.mode` changes. The conditional skip on `'view-status'` preserves both behaviors.

- [ ] **Step 3a: Update useFocusSync to accept and check focusZone**

In `frontend/src/hooks/useFocusSync.ts`:

```typescript
// Before (full function):
export function useFocusSync(
  state: PosFlowState,
  tab: string,
  setSearchText: (text: string) => void,
  setSearchFocusKey: React.Dispatch<React.SetStateAction<number>>,
  setFocusZone: React.Dispatch<React.SetStateAction<string>>,
) {
  useEffect(() => {
    // Synchronize focusZone with state mode when student selection state changes
    if (state.kind === 'student_selected') {
      setFocusZone('mode-' + state.mode);
    }
    // ...rest unchanged
  }, [state.kind, state.mode, tab, setSearchText, setSearchFocusKey, setFocusZone]);
}

// After:
export function useFocusSync(
  state: PosFlowState,
  tab: string,
  setSearchText: (text: string) => void,
  setSearchFocusKey: React.Dispatch<React.SetStateAction<number>>,
  setFocusZone: React.Dispatch<React.SetStateAction<string>>,
  focusZone: string,
) {
  useEffect(() => {
    // Synchronize focusZone with state mode when student selection state changes
    // Skip if focusZone is 'view-status' — RecentStrip click intentionally set it
    if (state.kind === 'student_selected' && focusZone !== 'view-status') {
      setFocusZone('mode-' + state.mode);
    }
    // ...rest unchanged
  }, [state.kind, state.mode, tab, setSearchText, setSearchFocusKey, setFocusZone, focusZone]);
}
```
