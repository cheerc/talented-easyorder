## §1.1: 搜尋框 Enter 取消聚焦 + 數字鍵自動聚焦

### 驗證步驟
1. 啟動 App → idle 首頁
2. 確認搜尋框（「輸入編號或姓名」）**預設不聚焦**
3. 按數字鍵（如 `1`）→ 確認自動聚焦到搜尋框，且 `1` 已填入
4. 按 `Enter`（搜尋框內容為空或無匹配學生時）→ 確認焦點離開搜尋框
5. 焦點離開後，按 `A` → 確認進入收入/支出模式

### 預期行為
- idle 首頁載入時，搜尋框不聚焦
- 無焦點時按數字鍵 → 自動聚焦 + 輸入該數字
- 搜尋框空白時按 Enter → blur 離開焦點
- 離開焦點後全局快捷鍵（A/Q/W/E）正常運作

### 預期行為

### 邊界條件

## §1.4: 收支流程 Escape/Enter 鍵處理 + ConfirmDialog 攔截

### 驗證步驟
1. 按 `A` 進入收支 → 輸入金額 → Enter → 進入類型選擇
2. 按 `Escape` → 確認返回 idle 首頁
3. 重新進入收支流程 → 完成一筆支出（金額 → 類型 → 原因 → 備註 → Enter）
4. 觀察 ConfirmDialog 是否正確攔截 Enter/Escape（不穿透到下層）

### 預期行為
- Escape 在收支流程任何步驟都能返回首頁
- ConfirmDialog 開啟時，Enter = 確認、Escape = 取消，不觸發下層快捷鍵
- 備註輸入後按 Enter，flash banner 跳出且不被 ConfirmBanner 立即 dismiss

## §2 — 數值輸入框限制

---

## §2.1: 所有數字輸入框僅接受正整數

### 驗證步驟

**收支金額框：**
1. 按 `A` 進入收支 → 在金額框嘗試輸入 `-`、`+`、`.`、`e`、`E`
2. 確認這些字元**無法輸入**
3. 輸入 `0` → 嘗試送出 → 確認被拒絕

**繳費金額框：**
4. 搜尋學生 → 選擇已訂過便當的學生（payment 模式）
5. 在繳費金額框嘗試輸入 `-`、`.`、`e` → 確認無法輸入

**改價金額框：**
6. 搜尋學生 → 在改價輸入框嘗試輸入 `-`、`.` → 確認無法輸入

**設定頁面：**
7. 進設定 → 便當單價輸入框嘗試輸入 `-`、`.` → 確認無法輸入
8. 每日開帳金額輸入框同上

**報表頁面：**
9. 進關帳流程 → 點算抽屜現金（countedCash）輸入框嘗試輸入 `-`、`.` → 確認無法輸入

### 預期行為
- 所有 6 個數字輸入框：`-`、`+`、`.`、`e`、`E` 被 onKeyDown 攔截
- 值為 0 或負數時拒絕送出
- 正常數字（1-9 開頭）可正常輸入和送出
